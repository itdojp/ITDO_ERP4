# 開発ロードマップ（ドラフト）

## 1. 全体方針

- **段階的移行**: 既存 Project-Open と並行稼働しながらモジュール単位で置き換え。
- **MVP 優先**: プロジェクト管理・タイムシート・請求（インボイス対応）・通知基盤を対象範囲とする。
- **MVP リリース目標**: 2025-12-10 に「動くもの」を提供する。
- **データ整合性重視**: PostgreSQL 15 へのスキーマ再設計と移行ツール整備を先行実施。
- **ドキュメント→実装サイクル**: 仕様整理（ITDO_ERP3）→抽出要約（ITDO_ERP4）→実装・検証の流れで進める。

## 3.1 Production Readiness Gate（2026-07時点）

Issue #1875 を正本として、PoC実装済み状態から「再現可能な検証結果を持つ本番候補」へ移行する。ここでは状態を以下に分けて管理する。

- **実装完了**: repo上のコード・ドキュメント・自動テストが存在する。
- **repo-side readiness完了**: 現在のcommitに対してCI/ローカルrunnerで再現可能に検証できる。
- **対象環境での運用完了**: 実環境の設定値、権限、rollback、backup/restore、外部製品現物を用いた確認が完了している。

直近の優先順は以下とする。

1. #1879 Roadmap/TODO/証跡索引/Draft PR #1818の現状同期。
2. #1877 frontend unit testの必須CI統合、#1878 deterministic data-quality blocking gate化。
3. #1880 → #1881 → #1882 の順で `integrations.ts` のservice境界を確立。
4. #1883 Release Candidate readiness runnerと証跡生成（repo-side runnerを標準入口化）。
5. #1426 ActionPolicy phase3_strictの対象環境trial/cutover/rollback、#544 S3 backup/restore、#1432 現物CSV回収をGo/No-Go依存として完了する。

#1430 は外部連携仕様全体の正本として継続管理する。#1432 が完了するまで、給与CSV仕様を推測で確定しない。
