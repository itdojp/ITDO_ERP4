# セキュリティ対策バックログ

## 目的
- リスク台帳に紐づく対策を **P0/P1/P2** で整理し、後続Issueの起点にする

## P0（最優先）
- 認証モード（header/hybrid）の公開制御を明文化し、起動時検証を追加（SR-001）
- チャット添付のAVスキャン運用を必須化（SR-005）
- 認可入口の集約方針（RBAC/ABAC）を設計し、影響範囲を明確化（SR-002）

## P1（重要）
- 依存関係監査（npm audit 等）のCIゲート化と運用ルール整備（SR-007）
- 監査ログのイベントスキーマ整備と記録範囲の拡張（SR-006）
- Webhook送信先の運用ルール/定期棚卸し（SR-004）
- PDF/添付の保存先設定のダブルチェック運用（SR-008）
- Secrets 管理（.env/ログマスキング）運用の明文化（SR-010）
- 入力検証（TypeBox/バリデータ）を主要APIへ拡張（SR-003）

## P2（改善）
- Push/Email 誤配信防止の導線整備（SR-009）

## 運用ルール（暫定）
- P0 は Issue 化してスプリントで追跡する
- P1/P2 は四半期単位で棚卸しして取り込み可否を決定する
