# 外部連携コード体系設計（給与/会計連携）

更新日: 2026-03-10
関連ISSUE: #1434, #1430

## 目的

- ERP4 内部 ID を外部連携キーとして露出させず、給料らくだ・経理上手くんα連携で使うコード体系を定義する。
- 将来のコード変更、組織変更、製品差し替えに耐える責任分界を明確にする。

## 前提

- 製品版差異と現物 CSV は `#1431`, `#1432` で確定する。
- 本書は 2026-03-10 時点の repository 実装を前提にした設計叩き台である。

## 1. 基本原則

- ERP4 内部 UUID は外部連携キーに使わない
- 外部システム向けコードは「意味を持つ安定コード」として別管理する
- コード値の責任主体を項目ごとに明確化する
- コード変更は履歴を残し、過去出力の再現を壊さない
- CSV 出力では「表示名」ではなく「コード値」を優先する

## 2. コード体系の責任分界

| コード種別          | 想定用途                | 責任主体       | 現状                            | 方針                                                 |
| ------------------- | ----------------------- | -------------- | ------------------------------- | ---------------------------------------------------- |
| 社員コード          | 給与・勤怠 CSV          | ERP4 人事/総務 | `UserAccount.employeeCode` 追加 | 給与前提 profile と責務を分離した                    |
| 顧客コード          | 売上/請求・補助科目候補 | ERP4           | `Customer.code` あり            | 既存コードを継続利用                                 |
| 業者コード          | 支払/仕入・補助科目候補 | ERP4           | `Vendor.code` あり              | 既存コードを継続利用                                 |
| 案件コード          | 部門/PJ/管理会計キー    | ERP4           | `Project.code` あり             | 既存コードを継続利用、会計部門コードとの関係は別管理 |
| 勘定科目コード      | 仕訳 CSV                | 会計責任者     | 未実装                          | ERP4 内に mapping master を持つ                      |
| 枝番/補助科目コード | 仕訳 CSV                | 会計責任者     | 未実装                          | 科目と分離した mapping master を持つ                 |
| 部門コード          | 仕訳/給与集計           | 管理部門       | 未実装                          | `department` 文字列とは別にコード管理が必要          |
| 税区分コード        | 仕訳 CSV                | 会計責任者     | 未実装                          | `taxRate` からの変換表を持つ                         |
| 証憑番号/伝票番号   | 仕訳・監査照合          | ERP4           | 一部あり                        | 文書番号を正本にし、連携伝票番号は派生規則で生成     |

## 3. 現行モデルでそのまま使えるコード

- `Customer.code`
- `Vendor.code`
- `Project.code`
- `Estimate.estimateNo`
- `Invoice.invoiceNo`
- `PurchaseOrder.poNo`

これらは既に一意制約があり、外部連携の参照キー候補として扱える。

## 4. 追加が必要なコード

### 4.1 社員コード

- 推奨
  - `UserAccount` とは別に、給与連携用途の `employeeCode` を明示的に保持する
- 理由
  - `externalId` は IdP/SCIM 連携用であり、給与システム連携キーと責務が異なる
  - `userName` は変更可能運用を既に想定しているため、給与連携キーには不適

### 4.2 部門コード

- 推奨
  - `department` 表示名とは別に `departmentCode` を導入する
- 理由
  - 給与・会計・管理会計で同じ表示名でもコード体系が異なる可能性がある
  - 表示名変更で CSV 互換性を壊さないため

### 4.3 勘定科目/枝番/税区分

- 推奨
  - ERP4 側に mapping master を置き、会計側コードを明示管理する
- 理由
  - `Expense.category`, `taxRate`, `Vendor`, `Project` だけでは仕訳 CSV の完成形を作れない
  - 変換責任を会計担当と ERP4 実装側で分離できる

## 5. 採番・変更ルール

### 5.1 社員コード

- 推奨
  - 既存の人事/給与運用で使っている社員番号があればそれを継承
  - 新規採番が必要な場合は連番でも良いが、表示名やメールアドレス由来のコードは使わない

### 5.2 証憑番号

- 推奨
  - `invoiceNo`, `poNo`, `estimateNo`, `vendorInvoiceNo` 等の既存番号を ERP4 正本とする
  - 会計連携用の伝票番号が別に必要な場合は、ERP4 正本番号を含む派生規則で生成する

### 5.3 コード変更時の履歴

- 推奨
  - 変更前コード、変更後コード、適用開始日、変更理由、変更者を保持する
  - 既出力ジョブは当時のコード値を再現できるよう snapshot 化する

## 6. 実装方針

### 6.1 マスタ保持

- 社員コードは `UserAccount.employeeCode` に持ち、給与前提項目は `EmployeePayrollProfile` に分離する
- 部門コード、会計コードは「表示名」と別カラムで持つ
- 連携ごとに意味が違う場合は、共通コードを無理に使わず mapping table で吸収する

### 6.2 出力時の優先順位

- 1. 明示設定された外部連携コード
- 2. 既存の stable code（`Customer.code`, `Vendor.code`, `Project.code` など）
- 3. 代替不可ならエラーにして出力停止

### 6.3 監査

- 出力ジョブごとに、使用したコード体系版と主要マッピングを追跡可能にする
- 手動上書きがある場合は、理由と変更者を AuditLog に残す
