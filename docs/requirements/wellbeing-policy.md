# ウェルビーイング入力ポリシー（ドラフト）

目的: 日報に付随する Good/Not Good/ヘルプ入力の扱い、閲覧権限、保管ポリシーを定義する。

## 権限
- 閲覧: 人事専用グループのみ。専用グループに人事グループを包含する運用とし、RBACで制御。
- 登録: 全ユーザーが自分の入力のみ登録/閲覧可能。
- 監査: 閲覧操作を監査ログに記録（誰が/いつ/どのレコード）。

## 収集項目（初期）
- `status`: good / not_good（必須、Good/Not Good）。未回答オプションを設ける場合は設定で許可。
- `job_engagement`: 0-4 の5段階（任意）
- `stress_level`: 0-4 の5段階（任意）
- `help_requested`: boolean（任意）
- `not_good_tags`: 配列（任意、not_good 時のみ表示）
  - 候補: 仕事量が多い / 仕事の進め方・役割 / 人間関係 / 体調 / 私生活 / 特になし・不明
- `notes`: 任意（短文、not_good 時の追加コメント）
- `entry_date`: 日付
- `user_id`: 入力者

## 保管ポリシー
- 方針: 法的に問題なければ保管期限を設けない（人事グループ限定閲覧の前提）。
- リスク対応: 個人情報・センシティブ情報に該当する可能性があるため、必要に応じて運用で保管期間を設定できるようフラグ/ジョブを用意する。
- 参考: 個人情報保護法上の安全管理は前提（アクセス制御、ログ、暗号化）。労働安全衛生法等で明確な保管年限が定まらないため、期限なし運用を採用し、今後法務判断が入った場合に変更する。

## 同意
- 初回ログイン時に「ウェルビーイング入力の利用目的・閲覧権限」を明示し、同意チェックを取得する。
- 同意の取得/撤回を監査ログに記録。

## ヘルプ / 相談ボタン（MVPに含める）
- 表示: ウェルビーイングモジュールの右上に常時表示。Not Good が続く場合は強調してもよい。
- モーダル選択肢（任意選択）: 上長に相談 / 人事に相談 / 産業医・保健スタッフ（いれば） / 社外相談窓口（EAP等） / 緊急の支援が必要かもしれない（公的窓口案内）。
- 各選択肢に通知先・対応目安を明記。
- 緊急時の案内を常時表示（医療機関や公的窓口への連絡を促す）。

## 表示/集計
- 個人向け: 直近のスコア推移（例: 2週〜1ヶ月）を表示。「評価には使わない」文言を明示。
- 管理者/人事向け: 5人以上のグループに限定した匿名集計（Not Good 割合、ストレス/充実感平均）を表示。個人特定を避ける。

## 表示/通知
- ダッシュボード: 人事グループのみ集計を表示（個別表示/集計の粒度は後続で決定）。
- 通知: 初期は通知なし。ヘルプリクエストに対し、人事グループへメール/タスク発行する拡張を後続検討。

## 今後の検討
- 保管期間を設定する場合のデフォルト年限（例: 1年/3年）と削除ジョブ。
- notes にセンシティブ情報が含まれるリスクへのフィルタリング/注意喚起。
- 集計の匿名化要件（必要ならフェーズ2以降）。
