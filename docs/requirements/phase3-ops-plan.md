# Phase 3 運用設計（ドラフト）

## 目的
監査ログ出力・アクセスレビュー・インサイト運用の実務フローを定義する。

## 監査ログ出力フロー
### 申請/承認
- 申請者: 管理部/監査担当
- 申請内容: 対象期間/対象データ/目的/提出先
- 承認者: 管理部責任者

### 実行/記録
- 実行者: 承認済みユーザのみ
- 出力形式: CSV/JSON
- 出力ログ: `audit_log_exported` を必須記録
- 保管: ローカル保管 + 外部保全（WORM）

### 監査対応
- 監査依頼→抽出→提出までのリードタイム目安を設定
- 依頼/提出履歴は運用ログとして記録

## アクセスレビュー
### 周期/責任
- 周期: 四半期
- 責任者: 管理部/人事

### レビュー対象
- ロール/グループ/案件割当
- 無効ユーザ/退職者の棚卸し

### 記録
- レビュー結果をCSV/議事録として保存
- 変更実施後に監査ログ確認

## インサイト運用
### 役割
- 閲覧者: 管理部/経営
- 監査: 閲覧ログを必ず記録

### 説明責任
- インサイトごとに根拠表示を義務化
- 誤検知時の対応フロー（確認→修正→記録）を定義

### 運用ルール
- 重要度の閾値設定
- 週次/月次でレビュー会議を実施

## 付帯ルール
- 監査ログ/インサイトの保存期間を明記
- 個人情報のマスキング/匿名化ルール
- 外部提出時の承認と送付先管理
