# Phase 3 運用設計（ドラフト）

## 目的
監査ログ出力・アクセスレビュー・インサイト運用の実務フローを定義する。

## 監査ログ出力フロー
### 申請/承認
- 申請者: 管理部/監査担当
- 申請内容: 対象期間/対象データ/目的/提出先/機密区分/提出期限
- 承認者: 管理部責任者（外部提出・高機密は情報管理責任者/担当役員の追加承認）

### 実行/記録
- 実行者: 承認済みユーザのみ
- 出力形式: CSV/JSON
- 出力ログ: `audit_log_exported` を必須記録
- 保管: ローカル保管 + 外部保全（WORM）

### 監査対応
- 監査依頼→抽出→提出までのリードタイムを明確化（通常: 3営業日以内、緊急: 24時間以内）
- 依頼/提出履歴は運用ログとして記録

## アクセスレビュー
### 周期/責任
- 周期: 四半期
- 責任者: 管理部（最終決裁）/人事（一次レビュー）
- エスカレーション: 管理部と人事で判断が割れた場合は管理部責任者が最終決定

### レビュー対象
- ロール/グループ/案件割当
- 無効ユーザ/退職者の棚卸し

### 記録
- レビュー結果をCSV/議事録として保存
- 変更実施後に監査ログ確認

## インサイト運用
### 役割
- 閲覧者: 管理部/経営
- 監査: 閲覧ログを必ず記録

### 説明責任
- インサイトごとに根拠表示を義務化
- 誤検知時の対応フロー（確認→修正→記録）を定義

### 運用ルール
- 重要度の閾値設定（例）
  - 重大（Critical）: 法令/契約違反、重大な不正兆候、複数案件への重大影響
  - 高（High）: 金額/工数の影響が大きく、短期間で重大化しうる兆候
  - 中（Medium）: 限定範囲への影響が見込まれるが即時重大損害ではない
  - 低（Low）: 参考情報レベル、影響範囲や影響度が小さい
- 週次/月次でレビュー会議を実施

## 付帯ルール
- 監査ログ: 原則7年間保存（社内データ保持ポリシーに準拠）
- インサイトデータ: 原則3年間保存（超過分は削除または匿名化）
- 個人情報のマスキング/匿名化ルール
  - 対象項目: 氏名/メールアドレス/電話番号/住所/社員ID/顧客IDなどの個人特定情報
  - マスキング（ログ/エクスポート）: 氏名は姓のみ、メールはローカル部の一部+ドメイン、電話は下4桁、住所は市区町村まで、各種IDは後半を `*` で置換
  - 匿名化（分析/インサイト）: 再識別可能なID/氏名/連絡先は削除または不可逆ハッシュ化
  - 例外: 正当理由+責任者承認のうえ、理由/承認者/閲覧者/期間を監査ログへ記録
- 外部提出時の承認と送付先管理
  - 承認レベル: 部門責任者 + 情報管理責任者（高機密は担当役員の追加承認）
  - 承認記録: 目的/範囲/送付先/提出手段をチケット等に記録
  - 送付先管理: 管理部がマスタ管理し、半期ごとに棚卸し
  - 非定型/新規送付先: 原則禁止、例外はリスク評価と一時/恒久登録の記録
  - 緊急時: 事前合意を最低限取得し、24時間以内に正式承認を補完
  - 送付方法: 組織で定めた安全なチャネル（暗号化メール/安全なファイル転送）に限定
