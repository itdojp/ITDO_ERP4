# ERP4 管理会計レポート要件定義（初期案）

関連 Issue: `#1446`, `#1430`, `#1433`, `#1434`, `#1435`, `#1439`, `#1440`, `#1441`

## 目的

- ERP4 上で管理会計に必要な初期レポートの範囲、利用者、集計軸、更新タイミングを定義する。
- 制度会計の確定値は外部会計システムに残しつつ、経営判断に必要な管理会計の速報値・内部管理値を ERP4 で扱う前提を明文化する。
- 既存の `project-profit` / `project-effort` / `overtime` / `delivery-due` / `burndown` / `project-evm` 実装を、管理会計用途でどう位置付けるかを整理する。

## 前提

- 案件損益の算出ロジックは `docs/requirements/profit-and-variance.md` を一次仕様とする。
- 自動配信・帳票フォーマットの基本は `docs/requirements/workflow-report-expansion.md` を参照する。
- 外部給与・会計連携の前提ギャップは `docs/requirements/erp4-payroll-accounting-gap-analysis.md` を参照する。
- 管理会計の初期フェーズでは、ERP4 内の原データから再現できる範囲を正本とし、制度会計確定値との差異は「速報値/管理値」であることを明示する。
- 初期フェーズでは、全社共通の間接費配賦は行わない。間接費は共通案件または疑似案件に計上し、必要な配賦ルールは後続フェーズで扱う。

## 想定利用者

| 利用者        | 主用途                                             | 必要な粒度                              | 既存実装で代替可能な範囲              | 追加要件                                     |
| ------------- | -------------------------------------------------- | --------------------------------------- | ------------------------------------- | -------------------------------------------- |
| 経営          | 月次の売上・原価・粗利・受注残の把握、赤字案件検知 | 月次、部門別、案件別                    | 案件別損益、納期未請求、工数予実      | 部門別損益、全社 KPI ダッシュボード          |
| 管理部        | 締め後の再現、配信、会計連携前確認                 | 月次、案件別、社員別、配信履歴          | CSV/PDF 配信、案件損益、残業          | 給与・会計連携前の照合、締めスナップショット |
| 現場責任者/PM | 担当案件の採算、工数超過、未請求確認               | 案件別、ユーザ別、グループ別、週次/月次 | 案件損益、工数予実、バーンダウン、EVM | ダッシュボード導線、権限別表示制御           |

## 現在 ERP4 で取得できるレポート

### 既存 API / UI / 配信

| 区分                   | API / 画面                                        | 用途                                 | 主な軸                            | 出力             |
| ---------------------- | ------------------------------------------------- | ------------------------------------ | --------------------------------- | ---------------- |
| 工数予実               | `GET /reports/project-effort/:projectId`          | 案件別の工数実績と経費実績           | project, from, to                 | JSON / CSV / PDF |
| 案件損益               | `GET /reports/project-profit/:projectId`          | 案件別の売上・直接原価・粗利         | project, from, to                 | JSON / CSV / PDF |
| 案件損益（ユーザ別）   | `GET /reports/project-profit/:projectId/by-user`  | 労務コスト比率配賦によるユーザ別採算 | project, userIds, from, to        | JSON / CSV / PDF |
| 案件損益（グループ別） | `GET /reports/project-profit/:projectId/by-group` | ユーザ集合単位の採算                 | project, userIds, label, from, to | JSON / CSV / PDF |
| グループ工数           | `GET /reports/group-effort`                       | 指定ユーザ集合の工数合計             | userIds, from, to                 | JSON / CSV / PDF |
| 個人別残業             | `GET /reports/overtime/:userId`                   | 個人別の工数合計                     | user, from, to                    | JSON / CSV / PDF |
| 納期未請求             | `GET /reports/delivery-due`                       | マイルストーン未請求管理             | project, from, to                 | JSON / CSV / PDF |
| バーンダウン           | `GET /reports/burndown/:projectId`                | ベースライン対比の消化管理           | project, baseline, from, to       | JSON             |
| EVM                    | `GET /reports/project-evm/:projectId`             | PV / EV / AC / SPI / CPI             | project, from, to                 | JSON             |

### 現状の強み

- 案件単位の売上・原価・工数・粗利は、ERP4 内データだけで再現可能である。
- CSV/PDF 出力と定期配信の土台が既に存在する。
- PM 向けの予実・採算確認という意味では、PoC/MVP のベースラインを満たしている。

### 現状の不足

- 部門別損益の正本データがない。
  - `department` は `UserAccount` にあるが、原価・売上の集計軸として正規化されていない。
- 給与原価の正本がない。
  - 現在の labor cost は `time_entries * rate_cards.unitPrice` の管理単価であり、給与確定値ではない。
- 制度会計向けの勘定科目別仕訳集計は未整備である。
- 月次締め後に再現可能な管理会計スナップショットがない。
- 権限別のダッシュボード要件が明文化されていない。

## 初期レポート一覧

### R1. 案件別採算サマリ

- 目的: 案件単位で売上、直接原価、粗利、粗利率、工数実績を確認する。
- 利用者: 経営、管理部、PM
- 集計軸:
  - project
  - from/to
- データソース:
  - `Invoice`
  - `VendorInvoice`
  - `Expense`
  - `TimeEntry`
  - `RateCard`
- 更新タイミング:
  - 画面はオンデマンド
  - 定期配信は月次/週次
- 実装状態:
  - 既存 `project-profit` で概ね対応済み
- 補足:
  - 速報値/管理値として扱う
  - 制度会計確定との差異は許容し、その差異原因は外部会計連携側で照合する

### R2. 案件別採算内訳（ユーザ別 / グループ別）

- 目的: 原価超過の原因と担当者/チーム別の負荷を把握する。
- 利用者: 経営、管理部、PM
- 集計軸:
  - project
  - user / user group
  - from/to
- データソース:
  - `project-profit-by-user`
  - `project-profit-by-group`
- 更新タイミング:
  - オンデマンド
  - 月次配信
- 実装状態:
  - 既存 API あり
- 要件補足:
  - 配賦方法 (`allocationMethod`) を必ず表示する
  - 意思決定用途では「労務コスト比率配賦」であることを明記する

### R3. 部門別損益

- 目的: 経営・管理部が部門単位の売上、原価、粗利を把握する。
- 利用者: 経営、管理部
- 集計軸:
  - department
  - month
  - 必要に応じて project drill-down
- 初期データソース方針:
  - 売上: `Invoice.projectId -> Project`
  - 直接原価: `VendorInvoice`, `Expense`, `TimeEntry`
  - 部門軸: 初期は `Project` の所属部門、または担当部門を正本候補とする
- 実装前提:
  - `department code` と `project -> department` の正規ルールが必要
  - `#1434`, `#1439`, `#1441` に依存
- フェーズ:
  - 初期フェーズでは要件定義のみ
  - 実装は後続

### R4. 月次 KPI ダッシュボード

- 目的: 月次の経営指標を 1 画面で把握する。
- 利用者: 経営、管理部
- 初期 KPI:
  - 売上実績
  - 直接原価
  - 粗利
  - 粗利率
  - 納期未請求件数/金額
  - 工数実績
  - 残業時間
  - 赤字案件数
- 集計粒度:
  - 月次
  - 全社 / 部門 / project
- データソース:
  - R1, `delivery-due`, `group-effort`, `overtime`
- 実装前提:
  - 既存レポート API の再利用を優先
  - KPI card / dashboard 表示は別 issue で設計

### R5. 管理部向け締め前照合レポート

- 目的: 給与・会計連携前に、ERP4 内の締め対象件数と未承認/未請求/未配賦項目を確認する。
- 利用者: 管理部
- 必要な確認項目:
  - 期間内の未承認 expense / vendor invoice / leave / time entry
  - 納期未請求
  - 単価未設定の工数
  - 外部コード未設定の master
- 実装前提:
  - `#1433`, `#1434`, `#1435`, `#1439`, `#1440`, `#1441` に依存
- フェーズ:
  - 初期は要件定義
  - 実装は後続

## 集計軸・データソース定義

### 売上

- 初期定義: `Invoice.totalAmount`
- 集計条件: `status in (approved, sent, paid)`
- 期間基準: `issueDate`
- 補足:
  - 入金基準は初期範囲外
  - 複数通貨は初期フェーズでは `Project.currency` と不一致のものを警告対象とする

### 原価

- 外注/仕入: `VendorInvoice.totalAmount`
- 直接経費: `Expense.amount`
- 労務原価: `TimeEntry.minutes * RateCard.unitPrice`
- 補足:
  - 労務原価は給与原価ではなく、管理単価ベース
  - 給与連携完成後は「給与確定値ベース原価」と「管理単価ベース原価」を併存させる可能性がある

### 部門

- 初期要件:
  - 部門別集計の正本は、外部連携コードを含む部門マスタで持つ
  - `UserAccount.department` の自由入力値をそのまま正本にはしない
- 依存:
  - `#1434` 外部連携コード体系
  - `#1439` ERP4 人事・給与前提マスタ整備

### PJ

- 初期要件:
  - 案件別採算は `Project.id` を正本とする
  - 表示/外部連携は `Project.code` を利用する

### 人件費配賦

- 初期方針:
  - 案件別採算では既存どおり `labor_cost` / `minutes` ベースの配賦
  - 部門別損益では配賦ルールを別途確定する
- 初期運用上の扱い:
  - 全社共通人件費は共通案件または管理案件へ集約
  - そこからの再配賦は後続フェーズ

## 更新タイミング

- オンデマンド API:
  - 案件別採算
  - 工数予実
  - バーンダウン
  - EVM
- 定期配信:
  - 月次の案件採算、未請求、残業、KPI サマリ
- 月次締め:
  - 初期は `PeriodLock` と運用記録で補完
  - 将来的には締めスナップショットを追加する

## 権限要件

- 既存レポート API は `admin` / `mgmt` のみ
- 初期フェーズの考え方:
  - 全社 KPI / 部門別損益: `admin`, `mgmt`
  - 自案件採算: PM / project manager に限定公開したいが、現状 route 設計とは差分がある
- 結論:
  - 初期要件として「PM が自案件のみ閲覧できる」要件を記録し、実装は後続検討とする

## KPI 一覧（初期案）

| KPI            | 定義                                  | 既存実装ソース                    | 実装状態                       |
| -------------- | ------------------------------------- | --------------------------------- | ------------------------------ |
| 月次売上       | 対象月の `Invoice.totalAmount` 合計   | `project-profit` 集計ロジック     | 一部流用可                     |
| 月次直接原価   | `VendorInvoice + Expense + laborCost` | `project-profit` 集計ロジック     | 一部流用可                     |
| 月次粗利       | 売上 - 直接原価                       | `project-profit` 集計ロジック     | 一部流用可                     |
| 粗利率         | 粗利 / 売上                           | `project-profit` 集計ロジック     | 一部流用可                     |
| 工数実績       | `TimeEntry.minutes` 合計              | `project-effort`, `group-effort`  | 実装済み                       |
| 残業時間       | `reportOvertime` の時間合計           | `overtime`                        | 実装済みだが定義見直し余地あり |
| 納期未請求件数 | `delivery-due` 件数                   | `delivery-due`                    | 実装済み                       |
| 赤字案件数     | 粗利 < 0 の案件数                     | `project-profit` の横断集計が必要 | 未実装                         |

## 初期フェーズの境界

### 初期フェーズで実装対象に含める

- 案件別採算とその配信
- 案件別採算のユーザ別 / グループ別内訳
- 工数予実、納期未請求、残業
- 管理会計ダッシュボードに必要な KPI 要件定義

### 初期フェーズで要件化のみ行い、実装を後続に回す

- 部門別損益
- 給与確定値ベースの人件費
- 勘定科目別集計
- 間接費の全社配賦
- 月次締めスナップショット
