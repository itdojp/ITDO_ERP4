# 外部 CSV artifact 回収依頼テンプレート

更新日: 2026-03-21
関連 Issue: `#1432`, `#1436`, `#1437`, `#1438`, `#1430`

## 目的

- 現行システム担当へ依頼すべき内容を、製品ごとに漏れなく整理する。
- `docs/requirements/external-csv-artifact-inventory.md` の未回収 artifact を、実際の依頼文へ落とし込む。
- repo 内 canonical sample と、実運用テンプレート原本の混同を防ぐ。

## 利用方法

- 実際の依頼は以下のテンプレートをそのまま転記するか、社内連絡手段に合わせて要約して用いる。
- 個人情報・機微情報を含む場合は、実データではなくマスキング版を依頼する。
- スクリーンショットよりも、CSV 実ファイル、ヘッダ付きテキスト、取り込み時のエラーメッセージを優先する。

## 給与らくだプロ向け依頼テンプレート

### 件名

ERP4 連携仕様確認のための 給与らくだプロ CSV テンプレート・取込条件ご提供依頼

### 本文

ERP4 から 給与らくだプロ Version 26.00 Rev.10.31 への連携仕様を確定したいため、現行運用で使用している CSV テンプレートと取込条件のご提供をお願いします。

対象

- 給与らくだプロ Version 26.00 Rev.10.31

依頼したいもの

1. 現行で実際に使用している CSV テンプレート
   - 社員マスタ CSV
   - 勤怠 CSV
   - マスキング済みの成功サンプル CSV
   - マスキング済みの失敗サンプル CSV（難しければ失敗時のエラーメッセージでも可）
   - ヘッダのみは暫定受領としては可。ただし artifact 回収完了とは扱わない
2. 各 CSV の取込仕様
   - 列名
   - 列順
   - 必須列
   - 任意列
   - 固定値列の有無
   - 使用しない列の扱い
   - 文字コード
   - 改行コード
   - 区切り文字
   - ダブルクォート要否
   - 日付形式
   - 時刻形式
   - 数値形式
   - 空値時の扱い
   - 文字数上限
   - 禁止文字
   - コード値制約
   - 重複時の扱い
   - 取込キー
   - 差分取込か全件取込か
   - 再取込時の扱い
   - エラー時の判定単位（ファイル全体/行単位）
3. 社員マスタ CSV について確認したい事項
   - 主キーとして使用している列
   - 社員コードの桁数、文字種、採番ルール
   - 雇用区分コードの値一覧
   - 所属/部門コードの値一覧
   - 役職コードの値一覧
   - 給与体系/締め区分/支払区分などの値一覧
   - 入社日、退職日、休職等の扱い
   - 振込口座情報の取込有無と必要列
4. 勤怠 CSV について確認したい事項
   - 打刻明細ではなく月次集計値を取り込む運用か
   - 必要な集計項目一覧
     - 出勤日数
     - 所定時間
     - 実働時間
     - 総残業時間
     - 法定内残業
     - 法定外残業
     - 深夜
     - 休日
     - 有給
     - 無給
     - 欠勤
     - 遅刻
     - 早退
     - その他必要項目
   - 締め済みデータの判定基準
   - 再締め/再出力時の扱い
   - 端数処理ルール
   - 時間単位（分、0.5h、10分単位等）
5. 運用情報
   - 現在の取込頻度
   - 全件/差分のどちらで運用しているか
   - 月次締め後の再取込があるか
   - エラー時の再送/再取込手順
   - 現行運用で困っている点や例外運用

補足

- 個人情報が含まれる場合はマスキング版で問題ありません。
- 実際に取り込めた CSV とエラーになった CSV の両方を依頼し、難しい場合でもエラーメッセージや取込結果画面を求めます。
- スクリーンショットではなく、できるだけ実ファイルまたはヘッダ付きテキスト形式での提供をお願いします。

特に先に確認したいもの

- 社員マスタ CSV の主キー列
- 勤怠 CSV の必要集計項目
- 文字コード、改行コード、時間単位
- 締め済みデータの判定基準
